# MathCoach-Agent：基于多智能体协作的数学解题与教学辅导系统设计

## 1. 系统定位

**MathCoach-Agent** 是一个面向数学学习场景的多智能体协作系统。系统将数学解题过程拆分为多个阶段，由不同 Agent 分别完成题目理解、解题规划、求解验证和教学讲解。

系统目标不是单纯给出答案，而是模拟一位“数学解题教练”，帮助用户理解题目、掌握方法、验证答案，并获得针对性的学习反馈。

---

## 2. 系统核心目标

系统主要实现以下目标：

1. **自动理解数学题目**
   - 识别题型；
   - 提取已知条件；
   - 判断涉及的数学知识点；
   - 明确求解目标。

2. **自动制定解题策略**
   - 根据题型选择合适的解题方法；
   - 给出清晰的解题步骤；
   - 提醒可能出现的易错点。

3. **完成数学求解与结果验证**
   - 生成完整解题过程；
   - 调用数学工具进行计算或验证；
   - 检查答案是否合理。

4. **生成教学化讲解**
   - 将解题过程转化为适合学生理解的语言；
   - 总结关键知识点；
   - 分析常见错误；
   - 生成相似练习题。

---

## 3. 系统整体架构

系统采用 4 个 Agent 的协作模式：

```text
用户输入数学题
        ↓
题目理解 Agent
        ↓
解题规划 Agent
        ↓
求解验证 Agent
        ↓
教学讲解 Agent
        ↓
最终输出：解题过程 + 验证结果 + 教学讲解 + 易错点 + 变式练习
````

系统整体流程可以理解为：



    读懂题目 → 制定方法 → 求解并验证 → 教学化讲解

## 4. Agent 设计

## 4.1 题目理解 Agent

### 角色定位

题目理解 Agent 负责对用户输入的数学题进行初步分析，将自然语言形式的题目转化为结构化信息。

它相当于系统中的“审题者”。

### 输入

用户输入的原始数学题，例如：


    求函数 f(x)=x^3-3x+1 在区间 [-2,2] 上的最大值和最小值。


### 核心功能

1. 判断数学题类型；
2. 提取题目中的已知条件；
3. 识别涉及的知识点；
4. 明确题目的求解目标；
5. 判断题目难度；
6. 输出结构化题目信息。

### 输出示例

```json
{
  "题型": "函数最值问题",
  "知识点": ["导数", "驻点", "闭区间最值"],
  "已知条件": {
    "函数": "f(x)=x^3-3x+1",
    "区间": "[-2,2]"
  },
  "求解目标": "求函数在指定闭区间上的最大值和最小值",
  "难度": "中等"
}
```

### 设计意义

题目理解 Agent 的作用是减少后续 Agent 的理解压力，使后续模块能够基于结构化信息进行推理和计算。

---

## 4.2 解题规划 Agent

### 角色定位

解题规划 Agent 负责根据题目理解 Agent 的输出，选择合适的解题方法，并生成解题路线。

它相当于系统中的“策略制定者”。

### 输入

题目理解 Agent 输出的结构化信息。

### 核心功能

1. 根据题型选择主要解法；
2. 生成分步骤解题计划；
3. 给出备用解法；
4. 标注关键步骤；
5. 提醒可能出现的常见错误。

### 输出示例

```json
{
  "推荐方法": "导数法",
  "解题步骤": [
    "对函数求导，得到 f'(x)",
    "令 f'(x)=0，求出驻点",
    "计算区间端点和驻点处的函数值",
    "比较函数值大小，得到最大值和最小值"
  ],
  "备用方法": "绘制函数图像辅助判断",
  "关键注意事项": [
    "闭区间最值问题必须比较端点",
    "不能只考虑导数为 0 的点"
  ]
}
```

### 设计意义

解题规划 Agent 使系统具备“先思考方法，再执行解题”的能力，避免直接生成答案导致的推理混乱。

---

## 4.3 求解验证 Agent

### 角色定位

求解验证 Agent 负责按照解题规划完成具体求解，并调用数学工具对结果进行验证。

它相当于系统中的“解题者”和“检查者”。

### 输入

1. 原始题目；
2. 题目理解 Agent 的分析结果；
3. 解题规划 Agent 的解题步骤。

### 核心功能

1. 生成详细解题过程；
2. 执行必要的代数计算、函数计算或概率计算；
3. 调用 Python、SymPy 等工具进行验证；
4. 检查答案是否与题目条件一致；
5. 给出验证结论和可信度评分。

### 输出示例

```json
{
  "解题过程": [
    "函数为 f(x)=x^3-3x+1",
    "求导得 f'(x)=3x^2-3=3(x^2-1)",
    "令 f'(x)=0，得 x=-1 或 x=1",
    "计算 f(-2)=-1, f(-1)=3, f(1)=-1, f(2)=3",
    "因此最大值为 3，最小值为 -1"
  ],
  "最终答案": {
    "最大值": 3,
    "最小值": -1
  },
  "验证方式": "符号计算与数值代入验证",
  "验证结果": "通过",
  "可信度": 0.96
}
```

### 工具调用方式：`Assertion` 原语

求解验证 Agent 在生成 `solution_steps` 与 `answer` 的同时，**额外输出一份 `assertions` 列表**——每条 assertion 是一个机器可校验的"声明"：

```json
"assertions": [
  {"expr": "diff(x**3 - 3*x + 1, x)", "expected": "3*x**2 - 3", "description": "求导"},
  {"expr": "solve(3*x**2 - 3, x)", "expected": [-1, 1], "description": "驻点"},
  {"expr": "(x**3 - 3*x + 1).subs(x, -1)", "expected": 3, "description": "f(-1)"},
  {"expr": "Max(-1, 3, -1, 3)", "expected": 3, "description": "最大值"}
]
```

每条 assertion 的语义：**LLM 声称 `expr` 在求值后等于 `expected`**。downstream 的 SymPy 验证器（`mathcoach.tools.sympy_verifier.verify`）独立计算每个 `expr`，与 `expected` 对账：

```python
# 概念性的 verifier 流程（详见 src/mathcoach/tools/sympy_verifier.py）
diff = simplify(parse(expr) - parse(expected))
if diff == 0:                   # symbolic 路径，confidence=0.95
    return passed
if abs(float(diff.evalf())) <= tol:  # numeric 路径，0.90
    return passed
# 当 assertion 含 free_vars（恒等式型）时，多次抽样：N=10，0.85
```

聚合规则：N 项 assertion 全 passed → overall passed；任一 failed → overall failed（含失败定位）；任一 error → overall error。**verifier 的判断会覆盖 LLM 自报的 `verification.confidence`**——任何 LLM 自封的 1.0 都拿不到外部信号背书前的最高分。

不可机器验证的题目（开放性证明、构造、解释类），LLM 应输出 `"assertions": []`，系统会把 LLM 自评 confidence 上限封顶到 0.6。

详细设计与字段约束见 [`openspec/changes/archive/...`](./openspec/) 中 `assertion-based-verifier` 的 design.md。

### 设计意义

求解验证 Agent 是系统可靠性的核心。**LLM 解题、SymPy 校验**，两者解耦：LLM 负责理解题面与策略选择，SymPy 负责对 LLM 写出的每一条数学声明做独立审计。这种"球员+裁判分离"是教练系统出口可信度的关键。

## 4.4 教学讲解 Agent

### 角色定位

教学讲解 Agent 负责将前面 Agent 的分析、规划和求解结果转化为适合学生理解的教学内容。

它相当于系统中的“数学老师”。

### 输入

1. 题目理解结果；
2. 解题规划结果；
3. 求解验证结果。

### 核心功能

1. 用通俗语言解释解题思路；
2. 总结本题涉及的核心知识点；
3. 分析常见错误；
4. 根据学生水平调整讲解风格；
5. 生成相似练习或变式题；
6. 给出学习建议。

### 输出示例

```json
{
  "教学讲解": "本题的关键是闭区间上的函数最值问题。对于闭区间上的连续函数，最大值和最小值可能出现在两类位置：区间端点，或者导数为 0 的驻点。因此，我们需要先求导找到驻点，再把端点和驻点的函数值全部算出来进行比较。",
  "核心知识点": [
    "导数的计算",
    "驻点的求法",
    "闭区间最值定理"
  ],
  "常见错误": [
    "只求导数为 0 的点，忘记比较区间端点",
    "求导计算错误",
    "找到驻点后没有代回原函数计算函数值"
  ],
  "变式练习": [
    "求函数 f(x)=x^3-6x^2+9x+1 在区间 [0,4] 上的最大值和最小值。",
    "求函数 f(x)=x^4-4x^2+2 在区间 [-2,2] 上的最大值和最小值。"
  ]
}
```

### 设计意义

教学讲解 Agent 使系统从“解题工具”升级为“学习辅助系统”。它不仅告诉用户答案，还帮助用户理解为什么这样做。

## 5. 系统运行流程

## 5.1 输入阶段

用户输入一道数学题，可以是自然语言形式，也可以包含数学表达式。

示例：

```text
求函数 f(x)=x^3-3x+1 在区间 [-2,2] 上的最大值和最小值。
```

---

## 5.2 题目分析阶段

题目理解 Agent 对题目进行结构化分析：

```text
题型：函数最值问题
知识点：导数、驻点、闭区间最值
已知条件：函数 f(x)=x^3-3x+1，区间 [-2,2]
求解目标：最大值和最小值
```

---

## 5.3 解题规划阶段

解题规划 Agent 根据题型生成解题方案：

```text
1. 求导；
2. 令导数为 0，求驻点；
3. 计算端点和驻点处的函数值；
4. 比较大小得到最值。
```

---

## 5.4 求解验证阶段

求解验证 Agent 按照规划完成求解：

```text
f'(x)=3x^2-3
令 f'(x)=0，得 x=-1, x=1

f(-2)=-1
f(-1)=3
f(1)=-1
f(2)=3

最大值为 3，最小值为 -1。
```

并通过工具验证：

```text
验证结果：通过
可信度：0.96
```

---

## 5.5 教学反馈阶段

教学讲解 Agent 生成面向学生的说明：

```text
本题的关键是：闭区间最值不仅可能出现在导数为 0 的点，也可能出现在区间端点。因此必须同时比较端点和驻点的函数值。
```

同时输出：

```text
核心知识点
常见错误
变式练习
学习建议
```

## 11. 系统亮点设计

## 11.1 多 Agent 协作

系统将数学解题过程拆分为多个阶段，每个 Agent 专注一个任务，避免单一大模型直接输出答案带来的混乱。

---

## 11.2 结构化中间结果

各 Agent 之间通过 JSON 格式传递信息，使流程更清晰，也便于调试和展示。

---

## 11.3 工具增强验证

系统不仅依赖大模型推理，还通过 Python 或 SymPy 进行数学验证，提高答案可靠性。

---

## 11.4 教学化输出

系统输出不只是最终答案，还包括解题思路、知识点总结、常见错误和变式练习，更符合数学学习场景。

---

## 11.5 可解释执行流程

用户可以看到每个 Agent 的中间结果，从而理解系统是如何一步步完成解题的。

---

## 12. 系统边界

为保证系统稳定性，初期版本可以设置以下边界：

1. 主要支持高中到大学低年级基础数学题；
2. 优先支持代数、函数、数列、概率等题型；
3. 暂不处理复杂几何图形题；
4. 暂不处理需要大量文本证明的高等数学证明题；
5. 暂不保证解决竞赛级数学难题；
6. 对超出能力范围的问题给出提示，并说明无法可靠求解。

## 14. 可扩展方向

在基础系统完成后，可以继续扩展：

1. **错题本功能**

   * 保存用户做错的题；
   * 统计高频错误知识点；
   * 自动生成复习计划。

2. **学生画像功能**

   * 记录学生水平；
   * 根据历史表现调整讲解难度。

3. **RAG 知识库功能**

   * 接入教材、课件或知识点文档；
   * 让教学讲解基于课程材料生成。

4. **多轮教学对话**

   * 学生可以继续追问；
   * 系统根据学生回答调整提示。

5. **图像题目输入**

   * 支持上传题目图片；
   * 通过 OCR 识别数学题。

6. **更复杂的验证机制**

   * 引入多种验证方法；
   * 对不同题型使用不同验证策略。

# 总结

MathCoach-Agent 采用 4 Agent 协作架构，将数学解题过程拆分为：

```text
题目理解 → 解题规划 → 求解验证 → 教学讲解
```

系统的核心优势在于：

```text
多智能体协作
结构化推理流程
数学工具验证
教学化反馈
过程可解释
```

该设计既能体现人工智能 Agent 系统的复杂性，又能控制实现难度，适合作为《人工智能应用开发》课程作业。
